# 高层建筑

工程互联网的三个项目之一

### 术语

####   驱动与响应

高层建筑在地震或风等影响下会发生变化(摇晃、震动、形变). 我们把这种影响叫做「驱动」,对应所产生的变化叫做「响应」.

####   固有频率

固有频率是物体的一种属性.

真实的建筑物可能有多阶固有频率, 称为一阶频率、二阶频率等

####   共振

当外界驱动频率与建筑固有频率相同时会发生共振, 导致建筑大幅度摇摆、断裂等, 所以我们要避免共振.

为了避免共振, 我们要使建筑物的固有频率避开风、地震等常见的频率区间.

因此,我们需要先知道如何测量建筑物的固有频率, 进而才能分析如何改变建筑物的固有频率,从而避免共振.

### 实验环境

####   高层建筑模型

本项目所采用的高层建筑模型高约2米, 外层塑料材质, 中间有一根杆支撑.

####   震动台

本项目采用的是单向震动台,仅能产生单方向的驱动输入.

####   传感器

总共有6个加速度传感器和2个应变传感器. 有五个加速度传感器分别置于建筑模型的不同高度,**朝向不同**;有一个加速度传感器与底部震动台紧密连接,可视作**震动台的实际驱动**.

每个加速度传感器输出x、y、z三个方向的加速度值, 每个应变传感器输出一个值.

本项目中的**响应**指的是建筑上的五个加速度传感器和两个应变传感器;**驱动**指的是震动台上的那一个加速度传感器.

**注意**: 每个加速度传感器的坐标系**并不一致**!因为加速度传感器的安装方向并不一致.

### 任务描述

本任务中, 我们的驱动有三种: 正弦波、真实地震波、无(自由振动)

**任务一:** 通过给定的驱动与响应数据集, 训练**模型**, 实现给定驱动预测响应、给定响应预测驱动.

**任务二:** 用手拉动建筑模型,然后突然释放做自由振动,根据响应数据计算固有频率、衰减速率.

**任务三:** 在高层建筑模型顶部加上**TMD**(调谐质量阻尼器), 实现上述两个任务.

注: 

	- 任务一的**响应预测**仅需预测顶部加速度传感器的值; **驱动预测**仅需考虑正弦波驱动,给出其**频率**即可.
	- 任务一的**模型**不一定是神经网络模型,可以用LSTM、也可以用有限元分析法,请根据数据集特征进行具体分析
	- 预测标准: 预测值与真实值的RMS

### 分析

####   数据集

数据集包含三类: 2条真实地震波、21条不同频率的简谐波、1条自由振动

数据集的结构: 时间 + (6*3+2)传感器值 	一共21列

采样频率: 50Hz

开头的一小段时间内,可能有部分传感器尚未启动,数值为空.

`raw_data`文件夹中所有`.csv`文件都是记录的传感数据。

#### 反思

此项目不适合使用神经网络架构, 

我们尝试了RNN、LSTM、Transformer、TCN

最后使用1dConv+biLSTM+MLP,但效果仍然不好.

问题如下:

​	1.因为数据集太小、噪声太多.

​	2.我们试图用驱动预测所有响应(3输入、17输出),无法拟合

​	3.各传感器坐标系不一致,FFT后观察到频域图很混乱,仅有个别传感器有突出的主频.

可能更好的解决方案是有限元分析.
